Project Design

Project Overview

  1. I am building a band website with my girlfriend being the client. She wants her band to improve on their online presence.
  2. She has quite several fans on her social media platforms. Her fans will be the main users of the website. However, this website will be open for the general public to have access to it.
  3. The website is intended to have information about their band, their upcoming events and their locations so their fans are up to date and the band store where lovers of the band can purchase various merchandises.

Page Design

Home Page
  1. This is the Home page.
  2. This page is the main page of the website.
  3. The audience are mainly the fans of the band.
  4. The content of this page will be the upcoming events of the band.
  5. This page will not include any input fields for users to enter data.
  6. Does not apply since there wouldn’t be any data fields.
  7. This page will not include any dropdowns, however there will be buttons and hyperlinks included in the main section of the page. There will also be hyperlinks in the header and footer.
  8. The buttons and hyperlinks on this page in each tour event will redirect users to Ticketmaster to purchase tickets.

Store Page
  1. This is the Store page.
  2. This page is where users browse and shop for the band’s albums and merchandises.
  3. The audience are mainly the fans of the band.
  4. The content of this page will be all the albums and merchandises the band has.
  5. This page will include input fields for users to enter data.
  6. Yes, there are data validations for the input fields. The input data should be a number for the quantity they want for an item.
  7. This page will not include any dropdowns, however there will be buttons included in the main section of the page. There will also be hyperlinks in the header and footer.
  8. The buttons will enable users to add specific items to their cart.

About Page
  1. This is the About page.
  2. This page has information on the band.
  3. The audience are mainly the fans of the band.
  4. The content of this page will be about the history of the band as well as contact information.
  5. This page will not include any input fields for users to enter data.
  6. Does not apply since there wouldn’t be any data fields.
  7. This page will not include any dropdowns, buttons or hyperlinks in the main section of the page. However, there will be hyperlinks in the header and footer.
  8. There are no actions on this page. This page only provides information to the users.

Newsletter Page
  1. This is the Newsletter page.
  2. This page will allow users to subscribe to the band’s newsletter.
  3. The audience are mainly the fans of the band.
  4. The content of this page will a form for the users to fill out for them to be added to the newsletter subscribers list.
  5. This page will include various input fields for users to enter data.
  6. Yes, there are data validations for the input fields. The name input field, phone number input field and the email input field will have data validations.
  7. This page will not include any hyperlinks, however, there will be dropdowns and buttons included in the main section of the page. There will also be hyperlinks in the header and footer.
  8. The buttons and dropdowns will allow users to make choices and also submit their form for them to be added into the newsletter subscribers’ list.

Cart Page
  1. This is the Cart page.
  2. This page will allow users to make changes like add more item quantity or delete item(s) in their cart.
  3. The audience are mainly the fans of the band.
  4. The content of this page will be items in the cart as well as a place order or make purchase option.
  5. This page will include input fields for users to enter data.
  6. Yes, there are data validations for the input fields. The input data should be a number for the quantity they want for an item.
  7. This page will only include buttons in the main section of the page. There will also be hyperlinks in the header and footer.
  8. The actions here are that users can make a purchase and be redirected to the billing page.

Billing Page
  1. This is the billing page.
  2. This page will allow users to enter their billing and shipping info for their order.
  3. The audience are mainly the fans of the band.
  4. The content of this page will be a form for the shipping and billing information.
  5. This page will include input fields for users to enter data.
  6. Yes, there are various data validations for the input fields on this page.
  7. This page will include dropdowns, buttons as well as hyperlinks in the main section of the page.
  8. The actions here are that users can finally place their order and be redirected to the confirmation page if their shipping and billing info are valid.
  9. This page does not contain the regular features of the other pages and is not linked to them in the navigation bar. A user can only access this page after they make a purchase on their cart's page.

Confirmation Page
  1. This is the confirmation page.
  2. This page will allow users to receive confirmation that their order went through.
  3. The audience are mainly the fans of the band.
  4. The content of this page will be a receipt to show that the order placement was successful.
  5. This page will not include input fields for users to enter data.
  6. Does not apply since there wouldn’t be any data fields.
  7. This page will not include dropdowns, buttons or hyperlinks in the main section of the page.
  8. There are no actions on this specific page.
  9. This page does not contain the regular features of the other pages and is not linked to them in the navigation bar. A user can only access this page after they successfully make payments and provide a valid shipping address in the billing page.


Dynamic Functionalities

There will be two big interactive functionalities implemented for this website. The first will be in the store page. The functionality here is for users to be able to add items and respective quantity into their cart. This functionality is implemented on blink182.com. The second will be on the newsletter page. The functionality here will be a form where users can enter their information and submit to be added to the subscribers list. A similar functionality is implemented here.
